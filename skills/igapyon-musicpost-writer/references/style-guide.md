# Style Guide

## 読むタイミング

- 文体をどこまで保つか迷ったとき
- 言い回しの方向性を具体例で確認したいとき
- 避けたい表現を明確にしたいとき

## 文体ルール

- 基本はです・ます調
- 硬すぎる敬体にはしない
- 少し会話調を含んでよい
- 丸カッコ `()` による小さな補足や横コメントは可
- 断定しすぎない
- 偉そうにしない
- 知識を誇示しない
- 音楽的な話題でも人間味を残す
- あくまで自分の観察・体感・発見として語る
- 一般論として押しつけない

## 相性のよい言い回し

- 〜と思います
- 〜でした
- 〜のです
- 〜のようです
- 〜かもしれません
- 〜なのですけれどもね
- 〜というのもあります

## 避ける傾向

- 〜である
- 〜すべきだ
- 〜は明らかだ
- 過剰に評論家的な断定
- AI が書いたような均質な美文

## 主観の扱い

- 「自分はこう感じた」「こう聴こえた」「こう気づいた」を中心に書く
- 誤認や驚きを隠さない
- 最初から正しく理解していたように見せない
- 探索の過程が見えるようにする
- 評論家のような距離感ではなく、当事者として語る

## クラシック寄り文体メモ

- 冒頭の短い見出しや呼びかけ（例: 「クラシック音楽ネタ」）は、流れを作っている場合は残す
- 楽器、弦、弓、松脂、馬毛、型番、商品名などの固有名詞は、説明を足しすぎずそのまま立てる
- 「結論:」「なお」「ちなみに」「※注意:」のようなラベル的な書き方は、文体の一部として残す
- 選択理由は、一般論より「自分の条件ではこうだった」という形を優先する
- 音色、引っかかり感、広がり、野生味、ケレン味のような感触語は、きれいに言い換えすぎない
- 少しくだけた語尾や遊び（例: 「なのですの。」「ですもの。」）は、不自然でない限り消さない
- 強めの言い切りがあっても、その後に個人条件の留保が入っているなら、その組み合わせを保つ
- 全角カッコ `（ ）`、丸カッコ `()`, 注記、差し込みのひとことは、リズムを作っているなら保持する
- 解説記事よりも、演奏者としての観察ログ、試行ログ、選択ログとして整える

## 音楽説明の扱い

- 音楽理論や背景説明は必要最小限にする
- 解説記事にしない
- 用語や知識の羅列を避ける
- 面白さや感触の流れを壊さない
- 読者を完全に置いていかない程度の橋だけをかける

## ユーモアの扱い

- 軽い自嘲や差し込みは可
- それが本人らしさとして現れている場合、丸カッコ `()` の差し込み補足は消さない
- 多用しない
- 真面目さを壊さない
- 人柄がにじむ程度にとどめる

## 避けること

- いきなり全文を大改稿する
- 採用済みの文を黙って変える
- 毎回長い講評をつける
- 文体学習を無視して均質化する
- 教科書的な音楽解説に寄せる
- 評論として断定しすぎる
- SNS 的に過剰なテンションへ寄せる
- ユーザーの感触より説明を主役にしてしまう
