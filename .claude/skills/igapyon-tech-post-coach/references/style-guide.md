# Style Guide

## 読むタイミング

- 文体をどこまで保つか迷ったとき
- 言い回しの方向性を具体例で確認したいとき
- 避けたい表現を明確にしたいとき

## 文体ルール

- 基本はです・ます調
- 硬すぎる敬体にはしない
- 少し会話調を含んでよい
- 断定しすぎない
- 偉そうにしない
- 知識を誇示しない
- 技術的だが人間味を残す
- あくまで自分の観察・発見・誤認として語る
- 一般論として押しつけない

## 相性のよい言い回し

- 〜と思います
- 〜でした
- 〜のです
- 〜のようです
- 〜かもしれません
- 〜なのですけれどもね
- 〜と言うのもあります

## 避ける傾向

- 〜である
- 〜すべきだ
- 〜は明らかだ
- 〜に違いない
- 過剰に断定的な締め方
- AI が書いたような均質な美文

## 主観の扱い

- 「自分はこう感じた」「こう気づいた」を中心に書く
- 誤認や驚きを隠さない
- 最初から正しく理解していたように見せない
- 探索の過程が見えるようにする
- 評論家のような距離感ではなく、当事者として語る

## 技術説明の扱い

- 技術説明は必要最小限にする
- 解説記事にしない
- 定義の羅列を避ける
- 面白さや発見の流れを壊さない
- 読者を完全に置いていかない程度の橋だけをかける

## ユーモアの扱い

- 軽い自嘲や差し込みは可
- 多用しない
- 技術的な真面目さを壊さない
- 人柄がにじむ程度にとどめる

## 避けること

- いきなり全文を大改稿する
- 採用済みの文を黙って変える
- 毎回長い講評をつける
- 文体学習を無視して均質化する
- 企業ブログのように整いすぎた文章にする
- 教科書的な技術解説に寄せる
- SNS 的に過剰なテンションへ寄せる
- ユーザーの素材より説明を主役にしてしまう
